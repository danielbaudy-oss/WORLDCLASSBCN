Por Qué el Cambio

El Real Decreto-ley 8/2019, con requisitos digitales actualizados para 2026, exige un sistema de fichaje digital que cumpla estándares técnicos específicos. El sistema anterior (Google Sheets) no cumplía varios de estos requisitos.

Cumplimiento Normativo

Sistema digital — PostgreSQL, no hoja de cálculo. Las hojas de cálculo no son conformes según la normativa actualizada.
Accesibilidad remota — Aplicación web accesible desde cualquier dispositivo con navegador.
Inalterabilidad — Triggers de PostgreSQL registran cada inserción, modificación y eliminación en una tabla de auditoría. No se puede eludir desde la aplicación.
GPS — Coordenadas capturadas en cada fichaje vía API de Geolocalización del navegador.
Acceso inmediato para inspectores — Consulta en tiempo real de todos los registros, correcciones y coordenadas.
Retención 4 años — PostgreSQL con backups automáticos, sin eliminación automática.
Empleado registra, responsable supervisa — Fichaje con cuenta Google, administración separada con permisos por rol.
Sin biometría — Autenticación Google OAuth, sin huellas ni reconocimiento facial.
En resumen: No existe certificación gubernamental obligatoria para fichaje en España. Lo que importa es cumplir los requisitos técnicos. WorldClass BCN los cumple todos: base de datos profesional, auditoría inalterable, GPS, acceso inspector en tiempo real y retención de 4 años.

Detalles Técnicos

Base de datos: PostgreSQL vía Supabase (UE)

Frontend: HTML/JS en GitHub Pages

Auth: Google OAuth

Auditoría: Triggers AFTER → audit_log

Datos: UTC, ISO 8601

GPS: Geolocation API por fichaje
